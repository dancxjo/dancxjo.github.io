Mechanical synthesis, neural speed

One current thread: use explicit linguistic machinery to manufacture the examples that small neural models can later run cheaply.

What we are building

Tongues is exploring a simple wager: if a slow, inspectable rule system can label text, split sentences, normalize words, infer pronunciations, and catch its own mistakes, then it can also become a teacher. The mechanical system generates rows. The neural model studies those rows until it can imitate the behavior at inference speed.

This is not just synthetic data in the usual sense. It is a curriculum made from the places where deterministic machinery breaks, disagrees, or exposes a useful edge case: abbreviations, quotations, sentence boundaries, function words, dialect tags, phoneme versus phone notation, and schema errors.

rule system → synthetic rows → verifier → failure cases ↘ ↙ small model learns the pattern

The goal is to distill careful linguistic scaffolding into compact models that are fast enough for streaming speech, repair, pronunciation, and agentic voice interfaces.

The ratchet

1. Build the slow teacher Sentence detectors, G2P helpers, Wiktionary parsers, normalizers, validators, and hand-written repair logic make behavior explicit.
2. Generate difficult examples The teacher creates ordinary rows and adversarial-ish rows: bad cuts, missing heads, malformed tags, language mismatches, and known pronunciation traps.
3. Train the neural approximation Small seq2seq models learn the input-output behavior without needing to run the whole symbolic apparatus at runtime.
4. Feed failures back in When the model or verifier finds a new weirdness, that weirdness becomes another specimen for the next training pass.

Clock laboratory

The original exhibit remains: decimal time and balanced quartovigesimal time, still ticking in the corner like numerological furniture.

Why clocks?

Because timekeeping is interface design with celestial consequences. Change the base, change the markings, change the words, and suddenly the day becomes a different machine.

Working principles

Not quite a manifesto. More like field notes from the workbench.

Systems should explain themselves

Logs, types, schemas, graphs, and interfaces should make the machine legible while it runs.

Memory is infrastructure

Agents need durable context, not just chat history. Recall belongs in the architecture, not as garnish.

Failure is curriculum

Contradictions, bad splits, malformed outputs, and verifier complaints are not just bugs. They are training specimens.
