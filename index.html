Mechanical synthesis, neural speed

A concrete AI pattern I keep returning to: build a slow, inspectable teacher, then distill its behavior into compact neural models.

What this demonstrates

Tongues uses deterministic linguistic machinery to label text, split sentences, normalize words, infer pronunciations, and catch its own mistakes. Those failures become training examples for smaller models that can run cheaply at inference time.

Netherwick applies a related engineering rhythm to embodiment: preserve captures, render the world, inspect failure, and let the system’s mistakes become fixtures for the next pass.

capture → projection → visible failure → fixture ↘ ↙ smaller, clearer systems learn the pattern

The goal is practical: compact models and legible systems for streaming speech, robot perception, repair, pronunciation, and agentic voice interfaces.

The ratchet

1. Build the slow teacherSentence detectors, G2P helpers, sensor pipelines, map renderers, normalizers, validators, and repair logic make behavior explicit.
2. Generate difficult examplesThe teacher creates ordinary rows and difficult cases: bad cuts, malformed tags, depth misalignment, stale robot modes, and known pronunciation traps.
3. Train the neural approximationSmall models learn input-output behavior without needing to run the whole symbolic apparatus at runtime.
4. Feed failures back inWhen the model, renderer, verifier, or robot finds a new irregularity, that irregularity becomes another specimen for the next training pass.

Working principles

Field notes from the workbench, minus the smoke machine.

Systems should explain themselves

Logs, types, schemas, graphs, captures, and interfaces should make the machine legible while it runs.

Memory is infrastructure

Agents need durable context, not just chat history. Recall belongs in the architecture, not as garnish.

Failure is curriculum

Contradictions, bad splits, depth errors, malformed outputs, and veto reasons are not just bugs. They are training specimens.
